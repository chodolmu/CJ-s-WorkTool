## 1. 기능별 베스트 레퍼런스

### 1-1. Discovery (요구사항 수집)

기존 도구 중 Discovery를 제대로 하는 곳이 거의 없음. 가장 가까운 건:

| 도구 | 방식 | 좋은 점 | 아쉬운 점 |
|------|------|---------|----------|
| **Copilot Workspace** | Issue → Spec → Plan → Code 순서 | 단계별 확인/수정 가능 | 텍스트 기반, 시각적이지 않음 |
| **Bolt.new** | 채팅으로 "뭘 만들지" 대화 | 즉시 결과 프리뷰 | 질문 안 함, 그냥 바로 만듦 |
| **Vercel v0** | 프롬프트 → 컴포넌트 생성 | 반복 수정 쉬움 | UI 컴포넌트만 가능 |
| **Vercel 프로젝트 셋업** | 단계별 위저드 (프레임워크 선택 등) | 깔끔한 스텝 UI | 코딩 전용, 요구사항 수집 아님 |

**채택 방향:**
- Copilot Workspace의 **단계별 확인/수정** 패턴
- Vercel 셋업의 **위저드 UI** (상단 진행 바 + 한 화면 한 질문)
- 우리만의 추가: **선택지 기반** 질문 + **스펙 카드** 요약

### 1-2. 에이전트 상태 대시보드

| 도구 | 방식 | 좋은 점 | 아쉬운 점 |
|------|------|---------|----------|
| **Cline** | VS Code 패널에 액션 로그 | 한 단계씩 승인/거부, 비용 추적 | IDE에 종속, 전체 프로젝트 뷰 없음 |
| **Devin** | 웹에서 에이전트 작업 실시간 관찰 | 에디터+터미널+브라우저 분할뷰 | 블랙박스 느낌, 비쌈 |
| **GitHub Actions** | 워크플로우 잡 매트릭스 | 병렬 작업 한눈에, 성공/실패 명확 | 실시간이 아님, 빌드 전용 |
| **Linear** | 이슈 상태 아이콘 + 보드/리스트 뷰 | 빠르고 직관적, 뷰 전환 | 에이전트가 아닌 사람 태스크 |
| **LangGraph Studio** | 그래프 노드 시각화 + 상태 인스펙션 | 에이전트 흐름을 시각적으로 이해 | 개발자 전용, UI 무겁 |
| **Datadog/Grafana** | 타일 기반 대시보드 + 신호등 상태 | "12 정상, 3 경고, 1 위험" 요약 | 모니터링 전용, 제어 불가 |

**채택할 베스트 조합:**
```
Devin의 실시간 관찰 화면
+ Linear의 상태 아이콘/뷰 전환
+ GitHub Actions의 병렬 잡 매트릭스
+ Cline의 비용 추적
+ Datadog의 집계 상태 요약 ("3 실행중, 2 완료, 1 오류")
```

### 1-4. 기억/컨텍스트 시스템

| 도구 | 방식 | 좋은 점 | 아쉬운 점 |
|------|------|---------|----------|
| **Claude Code** | CLAUDE.md 계층적 마크다운 | 심플, git 관리 가능, 사람이 읽기 쉬움 | 구조화 안 됨, 커지면 관리 어려움 |
| **Cursor** | .cursorrules + 코드베이스 임베딩(RAG) | 코드 맥락을 자동으로 찾아줌 | rules 파일은 단순 |
| **Devin** | VM 환경 자체가 기억 | 완벽한 지속성 | 무거움, 로컬 불가 |
| **CrewAI** | 단기/장기/엔티티/공유 메모리 (SQLite) | 에이전트 간 공유 O, 세션 간 지속 O | 설정 복잡 |
| **LangGraph** | 타입 상태 객체 + 체크포인트 (DB) | 타임트래블 가능, 상태 인스펙션 | 개발자가 스키마 정의 필요 |
| **AutoGen** | GroupChat 메시지 기록 = 공유 컨텍스트 | 간단, 모든 에이전트가 같은 기록 봄 | 기록이 길어지면 토큰 폭발 |

**채택할 베스트 조합:**
```
Claude Code의 마크다운 기반 (사람이 읽기 쉬움)
+ CrewAI의 구조화된 공유 메모리 (에이전트 간 보고서)
+ LangGraph의 체크포인트 (상태 스냅샷, 타임트래블)
```

**우리 도구의 3단계 기억 시스템:**
```
Layer 1: Guidelines (거의 안 바뀜)
  ├─ global_guidelines.md      ← 전체 프로젝트 규칙
  ├─ presets/game/rules.md     ← 프리셋별 규칙
  └─ agents/generator/role.md  ← 에이전트별 역할 지침
  → 매 에이전트 호출 시 해당하는 것만 선택 주입

Layer 2: Project State (누적 업데이트)
  ├─ project_state.json        ← 기능 목록, 완료/진행/대기 상태
  ├─ decisions.json            ← 주요 결정사항 기록
  └─ agent_reports/            ← 에이전트별 최종 보고서
  → 새 세션 시작 시 자동 주입

Layer 3: Session Logs (아카이브)
  └─ sessions/
      ├─ session_001.md        ← 세션 요약 (자동 생성)
      └─ ...
  → 필요할 때만 참조, 전체 읽기 금지
```

| 도구 | 방식 | 좋은 점 |

**채택: 모듈식 프롬프트 조립**
```

  base_instructions        (공통 규칙)
  + role_instructions      (에이전트 역할 — CrewAI YAML 방식)
  + project_state          (현재 프로젝트 상태)
  + task_specific          (이번 태스크 맥락)
  + output_format          (보고서 포맷)
```

### 1-6. Evaluator (자동 검증)

| 도구 | 방식 | 좋은 점 | 아쉬운 점 |
|------|------|---------|----------|
| **Anthropic 하네스** | Playwright MCP로 실제 브라우저 테스트 + 채점 | 가장 강력, 실제 앱 검증 | 복잡, 비쌈 ($200/run) |
| **Cline** | 빌드 성공 여부만 확인 | 간단 | 기능 검증 안 함 |
| **Bolt.new** | 실시간 프리뷰로 즉시 확인 | 빠른 피드백 | 자동 검증이 아닌 수동 |
| **Devin** | 자체 QA 루프 (테스트 코드 작성+실행) | 자율적 | 블랙박스 |

**채택: 단계적 Evaluator**
```
Level 1 (MVP): 빌드 통과 여부 + 타입 체크
Level 2: 별도 Evaluator 에이전트가 코드 리뷰
Level 3: Playwright로 실제 앱 테스트 (선택적)
```

### 1-7. 오케스트레이션 (에이전트 관리)

| 도구 | 방식 | 좋은 점 | 아쉬운 점 |
|------|------|---------|----------|
| **Anthropic 하네스** | Planner→Generator→Evaluator 3단계 자동 루프 | 검증된 패턴, 장시간 무인 실행 | 고정된 3역할 |
| **CrewAI** | YAML로 에이전트/태스크 정의, 순차/병렬 실행 | 유연한 구성, 위임 가능 | 코딩 필요 |
| **AutoGen** | GroupChat에서 에이전트들이 대화 | 자연스러운 협업 | 대화가 발산할 수 있음 |
| **LangGraph** | 그래프로 상태 머신 정의 | 가장 유연, 복잡한 흐름 가능 | 러닝커브 높음 |

**채택: Anthropic 하네스 기본 + 프리셋으로 확장**
```
기본 3역할 (항상):
  Planner    — Discovery 결과를 기술 스펙으로 확장
  Generator  — 스프린트 단위로 기능 구현
  Evaluator  — 결과물 검증, 통과/반려

프리셋에 따라 추가 역할:
  game/     → + BalanceTester, AssetGenerator
  webapp/   → + SecurityChecker, APITester
  custom/   → 사용자 정의 역할
```

## 5. MVP 기능 우선순위

### Must Have (v0.1)
1. Discovery 플로우 (선택지 질문 → 스펙 카드)
2. 에이전트 상태 대시보드 (실행/완료/오류)
3. 활동 타임라인 (실시간 로그)
4. 지침 자동 주입 시스템 (모듈식 프롬프트 조립)
5. Project State 자동 저장/불러오기
6. Evaluator 루프 (빌드 검증 → 재작업)
7. 코드 diff 프리뷰 + 승인/거부
8. 에이전트 완료/오류 데스크톱 알림
9. 스프린트 단위 진행 (중간 확인)

### Should Have (v0.2)
10. 프리셋 시스템 (게임/웹앱 프리셋)
11. 세션 요약 자동 생성
12. 토큰/비용 모니터링
13. 실시간 앱 프리뷰 (iframe)
14. 뷰 전환 (리스트/보드/타임라인)

### Nice to Have (v0.3+)
15. 커맨드 팔레트 (Cmd+K)
16. 프리셋 커스텀 에디터
17. 프로젝트 히스토리 타임트래블
