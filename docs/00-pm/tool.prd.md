## 1. Executive Summary

| Perspective | Description |
|-------------|-------------|
| **Problem** | Claude Code CLI는 비개발자/반개발자에게 진입장벽이 높고, 에이전트 팀 관리/프로젝트 기억 유지/계획 완전성 검증이 불가능하다. 기존 AI 코딩 도구(Cursor, Windsurf)는 개발자 중심이며, 비개발자를 위한 에이전트 오케스트레이션 데스크톱 앱은 시장에 부재하다. |
| **Solution** | Claude Code CLI를 감싸는 Electron 데스크톱 앱으로, 자연어 대화 기반 프로젝트 정의 -> AI 파이프라인 자동 실행 -> 시각적 관리 GUI를 제공한다. 기본 스킬/프리셋 번들링으로 즉시 사용 가능하며, 3-Tier 실행으로 토큰 비용을 최적화한다. |
| **Functional UX Effect** | Discovery 대화 -> 스펙 카드 -> 에이전트 팀 자동 구성 -> 3-Tier 실행(Direct/Light/Full) -> 실시간 오케스트레이션 시각화 -> 6단계 Phase Tracking -> 자동 학습 -> Git 관리 |
| **Core Value** | 하네스 엔지니어링의 코드화: CMD에서 수동으로 하던 에이전트 관리를 자동화/시각화하여 비개발자도 AI로 프로젝트를 완성할 수 있게 한다. 토큰 비용 최적화(3-Tier)로 cost-conscious 사용자에게 최적. |

### 5.2 Competitive Advantage

WorkTool의 핵심 차별점:
1. **유일한 비개발자 타겟 에이전트 오케스트레이션 앱**: Cursor/Windsurf는 개발자 전용, Bolt.new는 웹 전용 단순 도구
2. **멀티 에이전트 파이프라인**: 단일 에이전트가 아닌 Planner->Generator->Evaluator 체인 + 커스텀 에이전트
3. **3-Tier 토큰 최적화**: 단순 질문에 Full 파이프라인을 돌리지 않음 (비용 50%+ 절감)
4. **자동 학습**: 실패에서 배워서 같은 실수 반복 방지
5. **한국어 네이티브**: 영어 UI를 번역한 것이 아닌, 처음부터 한국어로 설계

### 5.3 Competitive Risks

| Risk | Mitigation |
|------|------------|
| Anthropic이 Claude Code Desktop을 강화하면? | WorkTool은 멀티 에이전트 오케스트레이션이 핵심 — Claude Desktop은 단일 세션 |
| Cursor/Windsurf가 비개발자 모드를 추가하면? | 이들은 IDE 기반이라 근본적으로 개발자 지향. Discovery -> Pipeline 플로우는 재현 어려움 |
| 오픈소스 Claude GUI 래퍼 경쟁? | 단순 래퍼는 많지만, 파이프라인+프리셋+학습 시스템은 없음 |

## 6. Market Sizing

### 6.1 TAM (Total Addressable Market)

- AI 코딩 도구 시장: **$10.12B** (2026, Precedence Research)
- No-Code AI 플랫폼 시장: **$6.2B** (2026, projected)
- 총 TAM: **~$16B** (AI 코딩 + No-Code AI 교집합 영역)

### 6.2 SAM (Serviceable Addressable Market)

- 비개발자/반개발자 중 AI 코딩에 관심 있는 세그먼트: TAM의 ~15%
- Claude Code 사용자 기반: 추정 200만+ 활성 사용자 (2026 Q1)
- 데스크톱 앱 선호 사용자: ~40%
- **SAM: ~$960M** (비개발자 AI 코딩 데스크톱 시장)

### 6.3 SOM (Serviceable Obtainable Market)

- 한국어 사용자 우선 타겟: SAM의 ~3% (한국 IT 시장 비중)
- 초기 오픈소스 침투율: ~0.5%
- Year 1 목표: **DAU 1,000, 월 프로젝트 생성 3,000건**
- **SOM: ~$1.4M** (한국 중심 초기 시장, 연간 환산)

- Secondary: YouTube (한국어 튜토리얼), 디시인사이드 프로그래밍갤
- Tertiary: Product Hunt, Hacker News (영문 확장 시)

**Metrics:**
- North Star: 프로젝트 완성률 (Discovery -> Building -> Completed 전환율)
- Activation: Discovery 시작 후 첫 Pipeline 실행까지의 시간 (<10분)
- Retention: 주간 프로젝트 활동률 (>30%)

**요구사항:**

- CheckpointModal에서 "누락 가능성 있는 기능" 표시

**구현 위치:** `pipeline.ts` > `runPlanner()` 이후, `requestCheckpoint()` 이전

**Acceptance Criteria:**
- Discovery에서 "일정 관리" 기능을 언급했는데 Planner가 빠뜨리면 경고 표시
- 사용자가 누락 항목을 "추가" 또는 "무시" 선택 가능

#### REQ-P0-06: E2E 엔진 테스트

**테스트 시나리오:**

### 9.2 Key Technical Decisions

| Decision | Rationale |
|----------|-----------|
| Electron + React | 크로스 플랫폼 데스크톱, 네이티브 파일시스템 접근, 기존 웹 에코시스템 활용 |
| better-sqlite3 | 서버 없는 로컬 DB, WAL 모드로 동시 읽기 성능 |
| Claude Code `--print --output-format json` | 비대화형 실행, 구조화된 출력 파싱 가능 |
| Zustand | 가벼운 상태 관리, React 특화 |
| 3-Tier (Direct/Light/Full) | 토큰 비용 최적화의 핵심 — 간단한 작업에 Full 파이프라인 불필요 |

### 9.3 Schema (SQLite v5)

- `projects`: 프로젝트 메타데이터 + spec_card_json
- `features`: 기능 단위 (순서, 상태, 일정)
- `agent_runs`: 에이전트 실행 기록 (토큰, 결과, 파일 변경)
- `activities`: 활동 로그 (실시간 스트리밍)
- `chat_messages`: 채팅 히스토리
- `sessions`: 세션 추적
- `learnings`: 에이전트 학습 기록
- `skills`: 스킬 (pattern + template)
- `phase_states`: 6단계 Phase 상태

## 10. Risk Assessment

| Risk | Probability | Impact | Mitigation |
|------|:---------:|:------:|------------|
| Claude Code CLI API 변경 | Medium | High | CLIBridge 추상화 레이어로 격리, 버전 핀닝 |
| 비개발자가 AI 결과물을 판단 못 함 | High | Medium | Evaluator 자동 검증 + 한국어 요약 + 점수 시스템 |
| 토큰 비용 폭주 | Medium | High | 3-Tier 자동 분류, 토큰 대시보드, 예산 경고 |
| Electron 앱 크기 (300MB+) | Low | Medium | 불필요한 의존성 제거, 점진적 로딩 |
| 경쟁사 비개발자 기능 추가 | Medium | Medium | 멀티 에이전트 + 프리셋 + 학습 = 진입장벽 높은 차별화 |
| Claude Code 가격 인상 | Medium | High | 모델 추상화 (향후 다른 LLM 지원 가능하게) |
