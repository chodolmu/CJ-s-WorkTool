WorkTool

AI Agent Team Management Desktop App

Electron React + TypeScript Claude Code CLI

Overview

WorkTool은 비개발자/반개발자가 AI 에이전트 팀을 시각적으로 관리하며 소프트웨어 프로젝트를 개발할 수 있는 데스크톱 앱입니다. 자연어로 프로젝트를 설명하면, AI가 자동으로 기획 → 코드 생성 → 검증까지 수행합니다.

핵심 컨셉

  • 대화형 Discovery — 질문지가 아닌 자유 대화로 프로젝트 정의
  • AI 에이전트 팀 — Planner, Generator, Evaluator + 커스텀 에이전트
  • 3-Tier 실행 모드 — 작업 크기에 따라 Direct / Light / Full 자동 선택
  • 자동 학습 — Evaluator 피드백을 축적하여 점점 나아지는 코드

기술 스택

  • Electron + electron-vite
  • React 18 + TypeScript
  • Tailwind CSS 3 + Framer Motion
  • SQLite (better-sqlite3)
  • Claude Code CLI (AI 엔진)
  • Zustand (상태 관리)

Installation

사전 요구사항

  • Node.js 18+
  • Claude Code CLI — 설치 가이드
  • Claude Code가 로그인된 상태 (claude --version으로 확인)

설치 및 실행

git clone https://github.com/chodolmu/CJ-s-WorkTool.git
cd CJ-s-WorkTool

npm install
npx electron-rebuild -f -w better-sqlite3   # 필수!

npm run dev                                   # 개발 모드 실행
⚠ 중요: npm install 후 반드시 npx electron-rebuild -f -w better-sqlite3를 실행해야 합니다. 안 하면 Electron과 Node.js의 모듈 버전 불일치로 에러가 발생합니다.

패키징 (Windows exe)

npm run package

dist/ 폴더에 설치파일이 생성됩니다.

사용 플로우

1

프로젝트 생성 — Discovery Chat

Dashboard에서 "+ New Project" 클릭 → AI와 대화하며 프로젝트를 설명합니다. "RPG 게임을 만들고 싶어" 같이 자연어로 말하면 됩니다. 상단에서 프로젝트 폴더 경로도 지정합니다.

2

Spec Review

AI가 대화 내용을 분석하여 Spec Card를 자동 생성합니다. 프로젝트 유형, 핵심 결정사항, 기술 스택, 확장 기능을 확인하고 수정할 수 있습니다.

3

Agent Team Setup

프로젝트 특성에 맞는 에이전트를 자동 추천받고, 필요한 에이전트만 골라서 팀을 구성합니다. 커스텀 에이전트도 직접 추가할 수 있습니다.

4

개발 — Chat 또는 Pipeline

Chat 탭에서 자연어로 작업을 요청하거나, Overview에서 "Start Pipeline" 버튼으로 전체 파이프라인을 실행합니다.

5

모니터링

Pipeline 탭에서 에이전트 플로우 시각화, Logs 탭에서 전체 활동 로그, Specs 탭에서 스펙 카드와 기능 진행률을 확인합니다.

실행 모드 (3-Tier)

Chat에서 메시지를 보낼 때, 작업 크기에 따라 자동으로 모드가 결정됩니다.

모드 AI 호출 적합한 작업 예시
⚡ Direct 1회 사소한 수정, 질문 "버튼 색상 바꿔줘", "이 코드 설명해줘"
🔧 Light 2회 중간 크기 기능 "비밀번호 확인 필드 추가", "페이지네이션"
🚀 Full 3+회 큰 기능/시스템 "소셜 로그인 시스템", "결제 모듈"

입력 바 왼쪽의 모드 버튼을 클릭하면 수동으로 모드를 전환할 수도 있습니다.

스마트 기능

🧠 Agent Learning

Evaluator가 반려할 때마다 피드백 패턴을 자동 추출하고 저장합니다. 다음 Generator 호출 시 축적된 교훈이 프롬프트에 자동 주입되어 같은 실수를 반복하지 않습니다.

🔄 Auto-Skill Detection

Generator가 만든 코드 패턴을 분석하여 반복되는 패턴(Page 컴포넌트, CRUD, Form 등)을 자동으로 감지하고 재사용 가능한 스킬로 등록합니다.

📝 Prompt Translator

사용자의 자연어 입력("버튼 색상 바꿔줘")을 AI 에이전트가 이해하기 좋은 구조화된 프롬프트로 자동 변환합니다. 의도 감지, 컨텍스트 주입, 출력 가이드를 포함합니다.

🎯 Smart Orchestrator

어떤 에이전트가 이 작업을 맡을지 자동 판단하고, 에이전트 간 결과를 전달하며, 실패 시 복구 전략을 결정합니다.

기본 스킬 (10종 내장)

프로젝트 생성 시 자동으로 활성화되는 보편적 개발 스킬입니다.

일정 관리

전체 프로젝트의 작업 일정을 시각적으로 관리합니다. PM(Planner)이 기능을 생성하면 일정이 자동 배분됩니다.

캘린더 뷰

월별 달력에서 각 일자별 작업을 확인합니다. 오늘 하이라이트, 월 이동, 상태별 색상 표시.

간트 차트 뷰

프로젝트별 피처 타임라인을 가로 바 차트로 표시합니다. 예정 바 vs 실제 진행 바를 비교할 수 있습니다.

자동 반영: Pipeline 실행 시 각 Feature의 예정 일정이 자동 배분되고, 실제 진행 시 시작/종료 시간이 자동 기록됩니다.

프로젝트 계획 자동 관리

Discovery 완료 시 Plan 문서가 자동 생성되고, 이후 모든 변경사항이 자동 추적됩니다.

  • 자동 생성 — Discovery 완료 → Spec Card 기반으로 Plan v1 생성
  • Feature 동기화 — Planner가 Feature 목록을 만들면 Plan에 자동 반영
  • 상태 추적 — Feature 상태 변경(대기→진행→완료) 시 Plan 자동 업데이트
  • 변경 이력 — 모든 변경을 changeLog에 시간순 기록
  • 스펙 일치도 — Discovery에서 언급한 기능이 실제 Feature 목록에 포함됐는지 % 표시
스펙-기능 교차 검증: Planner가 기능 목록을 만든 후, 스펙에서 언급됐지만 누락된 기능이 있으면 자동으로 경고가 표시됩니다. "일정 관리 만들겠다고 했잖아" 같은 문제를 방지합니다.

프로젝트 단계

6단계 프로젝트 라이프사이클을 관리합니다. 각 단계별 체크리스트와 게이트 조건이 있습니다.

단계설명게이트 조건
🔬 Research기술 조사, 아키텍처 탐색기술 스택 확정, 리스크 파악
📐 Design스펙 상세화, 데이터 모델기능 목록 확정, 데이터 구조 설계
🧪 Prototype핵심 기능만 빠르게 구현핵심 기능 동작, 방향 검증
🔨 Implement전체 기능 개발모든 기능 구현, 빌드 통과
🧪 Test검증, 버그 수정주요 버그 수정, 안정성 확인
✨ PolishUX 개선, 최적화릴리즈 준비 완료

알려진 제한사항

  • Claude Code CLI 필수 — AI 기능은 Claude Code가 설치되어 있어야 동작합니다.
  • npm install 후 반드시 npx electron-rebuild -f -w better-sqlite3 실행 필요
  • 모든 .tsx 파일에 import React from "react" 필요 (electron-vite 제약)
  • database.ts에서 require("better-sqlite3") 사용 — static import로 변경 금지
  • 앱 아이콘은 placeholder (16x16)
  • AI 엔진 E2E 테스트 미완 — Claude CLI 설치 환경에서 수동 검증 필요
